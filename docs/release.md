# Release / TestFlight setup

## What's already configured

- **App name**: `Giftsmith` (in `app.json`)
- **iOS bundle identifier**: `com.shatunov.giftsmith` — change in `app.json` → `expo.ios.bundleIdentifier` if you want a different ID
- **App icon**: `assets/images/icon.png` (1024×1024) — brass background, dark gift box silhouette with cream ribbons + bow
- **Splash**: midnight-green background (`#0F1A16`) with the brass gift glyph centered at 200pt
- **`eas.json`**: profiles for `development` (sim builds), `preview` (internal device builds), `production` (TestFlight + App Store builds)

The icon and splash images are regenerated by `node scripts/generate-icons.mjs` — edit the SVG inside that script if you want to iterate.

## One-time prerequisites (you do these manually)

1. **Apple Developer Program enrollment** ($99/year) — https://developer.apple.com/programs/
2. **Expo account** (free) — https://expo.dev/signup
3. **App Store Connect** — create the app record at https://appstoreconnect.apple.com once, before first submit. Note the **App Store Connect App ID** (numeric).

## First-time setup

Run from the project root. All of these are interactive (browser auth or terminal prompts) — they won't work via the AI sandbox.

```sh
# Authenticate with Expo (browser OAuth)
npx eas login

# Link this project to an EAS project. This adds an
# `extra.eas.projectId` field to app.json automatically.
npx eas init

# (Optional) sanity-check the eas.json this repo ships
npx eas build:configure
```

## First TestFlight build

```sh
# Production iOS build. First run prompts for Apple Developer
# credentials and generates a provisioning profile. Build runs on
# Expo's cloud — typically 10-20 minutes.
npx eas build --platform ios --profile production
```

When the build finishes, EAS prints a URL for the `.ipa` artifact. Then:

```sh
# Submit the build to TestFlight. EAS Submit auto-detects the App
# Store Connect record from the bundle identifier; if no record
# exists yet, it prompts you to create one. First run also walks
# you through Apple credential setup (App Store Connect API key,
# stored securely in EAS).
npx eas submit --platform ios --profile production
```

After Apple processes the build (10-30 min), you'll see it in TestFlight. Add yourself + any internal testers via App Store Connect → TestFlight tab.

## Iterating

For subsequent builds:

```sh
# Bump version automatically; previous builds stay on TestFlight.
npx eas build --platform ios --profile production
npx eas submit --platform ios --profile production
```

The `autoIncrement: true` in `eas.json` bumps the build number on each run; bump `expo.version` in `app.json` manually for user-facing version strings (e.g., 1.0.0 → 1.0.1).

## Local sim previews

To run on the iOS simulator without going through a build:

```sh
npx expo start --ios
```

To produce a `.app` you can drag-install on the simulator (no Apple Dev needed):

```sh
npx eas build --platform ios --profile development
```
